jika masih ada container mydb

FROM php:8.2-apache

Menjalankan container baru dengan Docker compose


1. Installasi aplikasi docker compose

sudo apt-get install docker-compose

2. pastikan tidak ada container yang berjalan

3. pindahkan file Dockerfile ke dalam folder app

pastikan kita berada di direktory /home/docker

cp aplikasi/Dockerfile aplikasi/app/Dockerfile 

Kita ubah direktory yang ada di dalam Dockerfile
nano aplikasi/app/Dockerfile

FROM php:8.2-apache
RUN docker-php-ext-install mysqli
COPY . /var/www/html

4. Membuat file docker-compose.yml didalam folder aplikasi

kita pastikan sudah berada didalam folder aplikasi

cd aplikasi

nano docker-compose.yml


version: "2"
services:
  mydb:
    image: mariadb:latest
    expose:
      - "3306"
    ports:
      - "3306:3306"
    restart: always
    hostname: mydb
    container_name: mydb
    environment:
      MARIADB_ROOT_PASSWORD: abc
      MARIADB_DATABASE: myapp
      MARIADB_USER: myappuser
      MARIADB_PASSWORD: abc
    volumes:
      - ./db:/var/lib/mysql
  app:
    build:
      context: ./app
      dockerfile: Dockerfile
    ports:
      - "8080:80"
    volumes:
      - ./app:/var/www/html
    restart: always

    depends_on:
      - mydb
    container_name: app
    hostname: app
  myadmin:
    image: phpmyadmin:latest
    ports:
      - "82:80"
    depends_on:
      - mydb
    restart: always
    environment:
      PMA_HOST: mydb
    container_name: myadmin
    hostname: myadmin

kita pastikan sudah berada didalam folder aplikasi

docker-compose up -d --build

F. Menggunakan Reverse Proxy pada Docker

1. membuat directory terlebih dahulu

Pastikan kita berada didalam diretory /home/docker

mkdir website
mkdir website/web-a
mkdir website/web-a/wp
mkdir website/web-a/db
mkdir website/web-b
mkdir website/web-b/wp
mkdir website/web-b/db

2. Membuat network untuk website

docker network create website

3. validasi dengan cara

docker network ls

4. Membuat file docker-compose.yml untuk container web-a dan web-b

Pastikan kita berada didalam direktory /home/docker

nano website/docker-compose.yml

version: '2'
services:
  nginx:
    image: nginxproxy/nginx-proxy:latest
    container_name: nginx
    ports:
      - "80:80"
    volumes:
      - /var/run/docker.sock:/tmp/docker.sock:ro
    environment:
      - TRUST_DOWNSTREAM_PROXY=false
    restart: always
    networks:
      - website
  db-a:
    image: mariadb:latest
    container_name: db-a
    environment:
      MARIADB_ROOT_PASSWORD: abc
      MARIADB_DATABASE: website
